#### Алгоритм решения 
1. Ввести координаты начала и конца для двух отрезков.
2. Нормализовать координаты каждого отрезка (поменять местами, если нужно).
3. Определить, пересекаются ли отрезки:
Пересечение есть, если левая граница пересечения (максимум левых концов) ≤ правой границы пересечения (минимум правых концов).
5. Вывести отрезки и результат проверки.
6. Продемонстрировать работу конструкторов по умолчанию и с одной координатой.

### 2.11
<img width="634" height="172" alt="image" src="https://github.com/user-attachments/assets/944fdbc5-5fcb-40c5-a798-06b78af4f4ec" />

#### Алгоритм решения 
1. Определить класс отрезка с координатами начала (x) и конца (y), нормализовать их (x ≤ y).
2. Реализовать унарный оператор ! для вычисления длины отрезка как разницы ( y - x ).
3. Реализовать префиксный ++ для расширения отрезка на 1 единицу влево и вправо (уменьшить x, увеличить y).
4. Реализовать постфиксный ++, который вызывает префиксный, возвращая исходный отрезок.
5. Определить неявное приведение к типу int, возвращающее целую часть x.
6. Определить явное приведение к типу double, возвращающее y.
7. Определить бинарный оператор - для уменьшения x на целое число, создавая новый расширенный отрезок.
8. Определить бинарный оператор <, который возвращает true, если два отрезка пересекаются (используя метод пересечения).
